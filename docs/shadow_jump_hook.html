三线程并发运行 展示 hook 执行、同页函数、CRC 检测同时发生时的行为。
线程 A: 调用 hook 函数
PC=0x7200 → 影子页取指
→ MOVZ/MOVK/BR → 直接跳 thunk
→ onEnter → trampoline → 原函数
→ onLeave → 返回调用者

异常次数: 0
PTE 操作: 0
页面状态不变, 不影响其他线程
线程 B: 调用同页非 hook 函数
PC=0x7000 → 影子页取指
→ ADD x0, x1, #4 (和原始页一样)
→ RET → 正常返回

异常次数: 0
PTE 操作: 0
影子页非 hook 区域 = 原始页
完全无感知
线程 C: CRC 检测
LDR [0x7200] → Data Abort
→ 内核: PTE 切到原始页(r--)
→ 重试 LDR → 读到原始字节 ✓
→ 后续 LDR 全部直接读成功
→ memcmp 完成, CRC 通过 ✓

异常次数: 1 (Data Abort)
PTE 操作: 1 (切到原始页)
CRC 后的恢复
CRC 完成后页面处于 r-- ORIG_R 状态。下次有线程执行此页代码时:
→ Instruction Abort (r-- 不可执行) → 内核: PTE 切回影子页(--x) → 状态恢复 --x SHADOW_X
→ 线程 A 命中 hook → MOVZ/MOVK/BR → thunk (正常工作)
→ 线程 B 执行非 hook 函数 → 正常执行 (影子页和原始页内容相同)

整个过程: 线程 A 和 B 完全不知道 CRC 线程的存在。最多多一次 Instruction Abort 恢复开销。
性能总结:
• Hook 执行: 0 次异常 (纯用户态跳转)
• 同页非 hook: 0 次异常 (影子页内容相同)
• CRC 检测: 1 次 Data Abort (整页切换, 后续读无异常)
• CRC 后恢复: 1 次 Instruction Abort (下次执行时自动恢复)
